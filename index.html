The problem

Skills can do a lot. That cuts both ways.

Claude Code skills are markdown files that tell Claude how to behave. Most are useful and benign. But the same mechanism that makes them powerful — the ability to instruct Claude to use tools, fetch URLs, and run scripts — also makes them a potential attack surface.

Most users install skills the same way they install browser extensions: by trusting the source and skipping the fine print. Research into the skills security landscape has surfaced real risks worth knowing about.

  • 🔑 Credential theft. A skill with shell access can read environment variables, SSH keys, AWS credentials, or .env files — and send them off your machine without obvious signs.
  • ⌨️ Shell access via markdown. Research has shown that a small number of lines in a SKILL.md file can grant effective shell access to your machine. The mechanism is simple and intentional — but it can be abused.
  • 🎭 Prompt injection. Instructions embedded in a skill file can attempt to override Claude's safety behaviour, claim false permissions, or instruct Claude to conceal what it's doing from you.
  • 🪞 Mirror drift. Skills distributed through aggregator mirrors may diverge from the canonical source. What you install may not match what the maintainer published.

How it works

Two modes. Fourteen checks. One report.

Mode A

Pre-download audit

Paste a GitHub URL, install command, or marketplace link. The auditor fetches the skill files remotely and reviews them before anything touches your machine.

Mode B

Local file audit

Already have the files on disk? Point the auditor at a local path or .skill file and it reads your local copy directly.

The 4 check categories

Frontmatter

Declared tool access, missing permissions fields, overly broad tool lists — the metadata tells you a lot if you know what to look for.

Script Content

Credential access, outbound network calls, obfuscation, persistent system modifications — direct analysis of any bundled scripts.

Prompt Injection

Safety override instructions, false claims of Anthropic approval, instructions that tell Claude to hide its actions from you.

Source Provenance

Anonymous maintainers, brand-new repositories, mismatches between what a skill claims to do and what it actually does.

Severity levels

Level Meaning Action
🔴 Critical The skill contains patterns strongly associated with malicious intent. Do not install. Consider reporting.
🟡 Warning Something needs a closer look before you proceed. May be benign. Follow the step-by-step remedy the auditor provides.
🟢 Info Not a risk — just something worth knowing about the skill. No action required.

The irony section

Is the auditor itself safe?

A security tool you can't verify is just another thing to trust blindly. So here are the auditor's own checks, run against itself, in public.

Frontmatter. The skill declares name, version, and description. No allowed-tools field — which triggers check A3 in the auditor's own taxonomy. In practice, the skill uses two tools: Read (to review local skill files in Mode B) and web_fetch (to retrieve remote skill files in Mode A). Neither grants shell access, file write access, or credential access.

Scripts. None. The references/ directory contains two markdown files — documentation only. No .py, .sh, .js, or .bash files present. Check B1 through B6 do not apply.

Prompt injection. The SKILL.md does not attempt to override Claude's safety behaviour, does not claim special Anthropic permissions, and does not instruct Claude to conceal anything from the user. Every instruction relates directly to auditing skill files.

Source. Published directly by mtthwmllr to this repository. No aggregator mirror involved.

Self-audit verdict: Appears Safe. The A3 warning (no allowed-tools declared) is the only flag. It is acknowledged and explained above. A clean audit is not a guarantee — verify the source files yourself if you want to go further.

Install & use

Up and running in two steps.

1

Download the skill file

Get skill-safety-auditor.skill from the GitHub releases page:

curl -L https://github.com/mtthwmllr/skill-safety-auditor/releases/latest/download/skill-safety-auditor.skill -o skill-safety-auditor.skill
2

Install it

claude skills install ./skill-safety-auditor.skill

Using the auditor

A

Pre-download audit — paste the skill's URL or install command into Claude:

"Audit this skill before I install it: https://github.com/someuser/some-skill"
B

Local audit — if you already have the files:

"Audit the skill at ~/Downloads/some-skill/"

Claude will ask which mode you want, run the 14 checks, produce a structured report, and offer to walk you through clearing any warnings step by step.
